Qué vas a lograr

Conectar GA4, Google Tag Manager y Search Console para medir UX (scrolls, clics, formularios), vigilar Core Web Vitals y crear un panel de decisiones en Looker Studio. Incluye snippets listos para copiar.

1) Instalar Google Analytics 4 (GA4)

Crea una propiedad GA4 y un flujo de datos Web. Copia el ID de medición (formato G-KXDYMQQVDM).

Snippet gtag.js (sin Tag Manager)

Colócalo en <head> de todas tus páginas públicas.

<!-- GA4 directo (solo si NO usas GTM) --> <script async src="https://www.googletagmanager.com/gtag/js?id=G-KXDYMQQVDM"></script> <script> window.dataLayer = window.dataLayer || []; function gtag(){dataLayer.push(arguments);} gtag('js', new Date()); gtag('config', 'G-KXDYMQQVDM'); </script>

3) Conectar Google Search Console

Verifica el dominio y envía el sitemap.xml. Controla indexación, Core Web Vitals y consultas orgánicas.

  1. Verifica por DNS (recomendado) o por meta (arriba en <head>).
  2. Entra a “Sitemaps” → añade https://mebe13.art/sitemap.xml (o el que uses).
  3. Revisa “Experiencia en la página” y corrige CWV (LCP, CLS, INP).
  4. En “Rendimiento” identifica keywords con impresiones altas y CTR bajo → mejora titles/meta.

4) Mapas de calor / Sesiones (Clarity o Hotjar)

Instálalo vía GTM. Observa scroll, clics erróneos y zonas ignoradas para priorizar cambios UX.

5) Plan de medición UX (mebe13.art)

Define qué acciones importan para negocio y UX. Empezamos con este set base.

  • Eventogenerate_lead: envío de formulario / clic en WhatsApp.
  • Eventoscroll_depth: 25/50/75/100 (engagement real).
  • Eventoselect_content: CTAs (ej. “Conversemos ahora”).
  • Dimensionespage_category, page_language.
  • Conversiones: define en GA4 (Admin → Eventos → Marcar como conversión).

6) Panel Looker Studio (1‑página)

  1. Conecta fuente GA4 (vista de eventos).
  2. KPIs: Usuarios, Sesiones, %Engagement, Leads, Tasa de conversión.
  3. Gráficas: Eventos por página, Scroll, Click‑map (por evento).
  4. Tabla: Consultas GSC → CTR vs Posición (para priorizar SEO).

7) Core Web Vitals (acciones rápidas)

  • LCP: Sirve imágenes .webp, precarga hero (<link rel=preload>), minimiza bloqueos.
  • CLS: Define width/height de imágenes y reserva espacio para fuentes.
  • INP: Diferir JS pesado, usar passive listeners, quitar librerías innecesarias.
  • Usa Lighthouse y el informe de “Experiencia” en GSC.

8) SEO‑UX checklist

  • Title (60‑65 chars) + Meta description con CTA.
  • Encabezados semánticos H1→H2→H3, un solo H1 por página.
  • Texto alternativo en imágenes clave (accesibilidad).
  • Enlaces con nombre descriptivo (no “haz clic aquí”).
  • Schema JSON‑LD (Organization/Person/LocalBusiness si aplica).
  • Mapa del sitio y robots.txt correctos.

9) Privacidad y consentimiento

Para cumplir normativas, muestra un aviso de cookies y política de privacidad. GTM puede disparar etiquetas tras consentimiento.

Ejemplo simple de banner (solo visual)

Checklist final (paso a paso)

  1. Crea propiedad GA4 y obtén G‑ID.
  2. Crea contenedor GTM y publica etiquetas (GA4 + Clarity).
  3. Verifica GSC (DNS o meta) y envía sitemap.
  4. Define eventos y conversiones (lead, scroll, CTAs).
  5. Arma el Looker dashboard 1‑página.
  6. Optimiza Core Web Vitals con PageSpeed y GSC.
